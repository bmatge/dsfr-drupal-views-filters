Composants de filtres exposés

Le même filtre peut s'afficher de trois façons. Zone primaire : un bouton replié qui ouvre un panneau. Zone secondaire : l'input déplié, posé directement sur la page, sans modale. Zone tertiaire : un récapitulatif en tags retirables. L'enjeu de mise en page (bloc / flex / inline) se concentre sur la zone secondaire.

Zone secondaire — input déplié en direct

Aucun panneau : le contrôle complet est posé sur la page. Deux décisions de layout indépendantes.

Décision 1 — orientation du groupe

En ligne (flex) : label et contrôle côte à côte. Réservé aux contrôles compacts — toggle, ou radios/cases à libellés très courts.

En bloc : label au-dessus, contrôle dessous. Pour les champs (texte, date) et toute liste de cases.

Décision 2 — disposition des valeurs

Inline-wrap : plusieurs cases/radios par ligne, passage à la ligne automatique. Si tous les libellés ≤ 20 caractères.

Liste verticale : une case par ligne. Si un libellé > 20 car., ou > 10 options (+ recherche & scroll).

Zone tertiaire — récap en tags

Alternative à la primaire pour exposer les valeurs choisies. Conteneur toujours flex-wrap. Le label passe en flex-basis:100% (tags en bloc dessous) si ≥ 5 valeurs ou un libellé > 28 car. ; sinon inline.

Règles d'arbitrage (pseudo-code)

Seuils paramétrables : SEUIL_LIBELLE_LONG = 20 (zone secondaire), MAX_OPTIONS_LISTE = 10, SEUIL_TERTIAIRE_BLOC = 28.

Zone secondaire — input direct

// Décision 1 — orientation du groupe
groupe_en_ligne = type in {booléen}
  OU (type in {mono,multi}
      ET nb_options <= 3
      ET max(longueur) <= 20)
// sinon → groupe en bloc (label au-dessus)

// Décision 2 — disposition des valeurs
valeurs_verticales = max(longueur) > 20
  OU nb_options > 10
// sinon → inline-wrap

Zone tertiaire — tags récap

tertiaire_en_bloc = type == "multi"
  ET (nb_valeurs >= 5
      OU max(longueur) > 28)
// sinon → inline

/* CSS commun */
.groupe { display:flex; flex-wrap:wrap;
          align-items:baseline; gap:8px; }
.groupe--bloc .label { flex-basis:100%; }
.valeurs--vertical { flex-direction:column; }

En résumé pour le dev

  • Primaire = bouton : compteur (n) pour le multi, nom de la valeur pour le mono.
  • Secondaire = input direct. Deux décisions séparées : orientation du groupe (ligne/bloc) et disposition des valeurs (inline-wrap / vertical). Un multi à libellés courts → cases inline-wrap qui passent à la ligne.
  • Tertiaire = tags récap, conteneur flex-wrap, bascule bloc via flex-basis:100% sur le label.
  • Un seul composant Twig paramétré par type, options, zone couvre les trois affichages.
